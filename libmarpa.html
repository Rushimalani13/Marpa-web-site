About Libmarpa

Libmarpa is the C language library that implements the core of Marpa. It is part of Marpa::R2, a Perl CPAN module. For the moment, this entanglement makes sense -- all of Libmarpa's tests are in Perl, and you really don't want to install Libmarpa without testing it on your configuration.

Installing Libmarpa

Libmarpa's only requirements are Perl 5.10 and a C90 compiler. To install Libmarpa:

  • Download, install and test Marpa::R2. Please do not skip the testing.

  • In the Marpa::R2 distribution file will be the file Marpa-R2-N.NNNNNN/r2/libmarpa.tar, where N.NNNNNN is your Marpa::R2 version number. Assuming that your Marpa::R2 distribution came as a tar file, that means the Libmarpa comes as a tar file inside a tar file.

  • The Libmarpa tar file is a product of the GNU autotools. I went to some trouble to keep it very "vanilla", so that it should have all the standard targets and install in all the usual ways.

  • The Libmarpa API document is online as HTML. PDF's of the Libmarpa API docs for the most recent versions can be found on Github's Marpa::R2 downloads page. Sources for the Libmarpa documentation are included in the Marpa::R2 distribution.

The name of the tar file

Some may have noted, on reading my claim that libmarpa.tar is a very standard GNU autotools distribution file, that its name is not GNU autotools standard -- it is not a compressed file, and it does not have the version number as part of its name. libmarpa.tar is not compressed because currently it is delivered inside another tar file, one which is almost always compressed. And the name is version-less for the convenience of the Perl build software.

The contents of libmarpa.tar are completely autotools standard, so that if you want a more conventionally named distribution file, the shell command "./configure; make dist" will produce one.

Libmarpa and Perl

Libmarpa can run and even build on a system without Perl. It has no runtime, build-time or compile-time dependence on Perl. Testing, however, does require Perl and I strongly discourage running Libmarpa in a configuration on which it has not been tested.

Libmarpa internals

The source files for Libmarpa come as part of the Marpa::R2 distribution, and can also be found in the Marpa::R2 github repository. I worked hard to limit the prerequisites needed to configure, compile and run Libmarpa. Working on Libmarpa's internals is another matter. Libmarpa is literate programming -- and work on Libmarpa's internals requires, among other things, Knuth's CWeb and therefore Tex.

Libmarpa is highly mathematical code, and its internals are, frankly, quite daunting. As an algorithm, Marpa has its moments of beauty, but nobody can say it achieves that beauty through simplicity. I once had hopes that Marpa would have the powerful, simple beauty of a late Picasso. But Angkor Wat and Hieronymous Bosch have their own beauty.
